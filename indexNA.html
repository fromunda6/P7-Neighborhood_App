Main Features, or the "What"-the "How" is constrained by, but not precisely defined by, the "what":

  1. by Default, user will be presented with all[5] locations in a listView and on map, the latter being animated on interact
  2. a text input box, wired up to the listView, that will filter locations appearing on listView and MAP
  3. location-specific info on click of map markers will be based on both pers knowledge(read:hard-coded) and as derived from a 3pAPI
  4. error handling of some sort, asynchronously made api requests, device-responsive design
